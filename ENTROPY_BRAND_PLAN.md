# 熵导图 · 品牌与交互改造计划

## 核心理念

- **产品名**：熵导图（Entropy Map）
- **信息熵**：追问 = 熵增（增加不确定性、发散）；回答 + Tips = 熵减（增加确定性、收敛）
- **右键能力归类**：
  - **熵增方案**：追问（后续可扩展）
  - **熵减方案**：人工熵减（回答）、智能熵减（Tips）

基础功能（回答、追问、Tips 的后端与业务逻辑）**不变**，仅做品牌包装与交互形态升级。

## Phase 1：品牌露出

| 步骤 | 内容 |
|------|------|
| 1.1 | 页面 `<title>` 改为「熵导图」或「熵导图 - MindBridge」 |
| 1.2 | 侧边栏等关键文案出现「熵导图」品牌名 |
| 1.3 | 顶栏/英雄区等可考虑副标题或 slogan（如「控制信息的熵增与熵减」） |

---

## Phase 2：右键菜单结构重组

| 步骤 | 内容 |
|------|------|
| 2.1 | 右键仅显示 **两个** 一级项：**熵增方案**、**熵减方案** |
| 2.2 | **熵增方案**：鼠标悬停展开子项 → **追问**（后续可加更多熵增能力） |
| 2.3 | **熵减方案**：鼠标悬停展开子项 → **人工熵减**（= 回答）、**智能熵减**（= Tips） |
| 2.4 | 子项点击仍触发原有逻辑：`answer` / `spawn` / `tips`（以及 section 的「在本板块下生成新问题」） |
| 2.5 | 根据节点类型（section vs 普通）继续隐藏/显示对应子项（如 section 只显示追问类） |

---

## Phase 3：圆盘菜单形态

| 步骤 | 内容 |
|------|------|
| 3.1 | 菜单从「竖列方格」改为 **以鼠标（或玻璃光标）为圆心的圆盘** |
| 3.2 | 两个主项分布在圆盘上（如左右对称，或上下弧），围绕圆心 |
| 3.3 | 悬停某一主项时，其子项在该侧 **扇形/弧段** 内展开，或小圆盘形式 |
| 3.4 | 圆盘采用玻璃质感，与现有 `#glass-cursor` 风格统一；可略大一圈，视觉上「包围」光标 |

---

## Phase 4：Logo 与视觉

| 步骤 | 内容 |
|------|------|
| 4.1 | **熵增方案**：特色 logo（非加号）— 意象：发散、分支、追问（如多向箭头、树枝、问号云） |
| 4.2 | **熵减方案**：特色 logo（非减号）— 意象：收敛、聚焦、确定（如聚拢线、靶心、灯泡） |
| 4.3 | **追问**：小 logo（如问号+分支、或对话气泡） |
| 4.4 | **人工熵减**：小 logo（如笔/手写、用户） |
| 4.5 | **智能熵减**：小 logo（如星星/魔法棒、AI） |
| 4.6 | 菜单整体配色：熵增偏暖/橙红、熵减偏冷/蓝绿，与功能语义一致 |

---

## Phase 5：JS 逻辑

| 步骤 | 内容 |
|------|------|
| 5.1 | `openNodeContextMenu(x, y, nodeId)`：菜单中心定位到 `(x, y)`，渲染圆盘 |
| 5.2 | 圆盘半径 + 子项展开区域做视口边界检测，避免超出屏幕 |
| 5.3 | 两大项 hover 显示对应子项；点击子项调用现有 `handleContextMenuClick(action)` |
| 5.4 | 保留 section 节点时的显示逻辑（只显示追问；追问文案为「在本板块下生成新问题」） |

---

## Phase 6：主题与贯穿

| 步骤 | 内容 |
|------|------|
| 6.1 | 黑客松主题下，圆盘菜单使用主题色（如 cyan/绿/紫）与玻璃底 |
| 6.2 | 其它界面文案可保留「追问」「Tips」「回答」，必要时加「熵增/熵减」辅助说明，使熵品牌贯穿 |

---

## 实现顺序建议

1. **Phase 1** → 2 → 3 → 4 → 5 → 6  
2. 其中 Phase 2+3 可先做「结构 + 圆盘布局」，再在 Phase 4 贴 Logo 与配色，Phase 5 贯穿 JS。

## 实现状态（已落地）

- **Phase 1**：title 改为「熵导图 · 控制信息的熵增与熵减」，侧边栏文案「熵导图 · 当前为黑客松竞赛演示」。
- **Phase 2–3**：右键仅两大项「熵增方案」「熵减方案」，悬停展开子项；圆盘玻璃菜单，圆心对准右键位置，边界 clamping。
- **Phase 4**：熵增主 icon 为分支/发散，熵减主 icon 为灯泡；子项追问/人工熵减/智能熵减各有独立小 icon。
- **Phase 5**：`openNodeContextMenu` 圆心定位与边界检测，section 节点仅显示熵增（追问→「在本板块下生成新问题」），子项点击仍走 `handleContextMenuClick(action)`。
- **Phase 6**：黑客松主题下圆盘与子菜单使用主题色（熵增紫、熵减青）。
