# 黑客松测试用策划案：智能健身镜

## 一、项目概述

我们计划做一款**智能健身镜**产品，面向家庭场景。用户站在镜子前即可跟随 AI 教练完成动作识别与纠正，无需额外穿戴设备。目标是在 48 小时黑客松内完成可演示的 MVP：实时姿态检测 + 3～5 个标准动作的跟练与纠错。

## 二、问题与机会

- **问题**：居家健身缺乏专业指导，动作错误易导致损伤；线下私教成本高、时间不灵活。
- **机会**：大屏与摄像头普及，计算机视觉成熟，用户对「无接触、在家练」需求明显上升。

## 三、解决方案

- 镜面显示 AI 虚拟教练与标准动作示范。
- 通过摄像头做实时骨骼关键点检测，对比用户动作与标准动作，给出语音/文字纠错提示。
- MVP 先支持「深蹲、开合跳、平板支撑、高抬腿、弓步蹲」五类动作，每类 1～2 个难度档位。

## 四、技术实现

- **前端**：大屏 Web 或 Android 镜端 UI，低延迟视频流。
- **姿态估计**：优先采用 MediaPipe 或轻量 OpenPose，保证 30fps 以上。
- **动作匹配与纠错**：预存标准动作关键点序列，实时计算用户序列与标准的相似度，低于阈值时触发提示。
- **部署**：本地边缘设备（如树莓派 + 摄像头）或云端 API 二选一，MVP 先本地。

## 五、商业模式

- 硬件：与镜面厂商合作，预装或后装镜端系统。
- 软件：免费基础跟练 + 付费课程包（专业教练录制课程、进阶计划）。
- 后续可拓展 B 端（健身房、酒店）镜端授权。

## 六、团队与执行

- 角色：1 产品/协调、1 前端、1 算法/后端；共 3 人，48 小时内分工明确。
- 里程碑：Day1 上午完成环境与姿态 demo，下午完成 2 个动作的跟练闭环；Day2 上午补全 5 个动作与基础 UI，下午联调、录屏与答辩准备。

## 七、风险与应对

- **风险**：光线或角度导致姿态识别不稳定。
- **应对**：MVP 限定「正面、上半身入镜」场景，并在 UI 上提示用户站位；后续迭代再扩展多角度。
- **风险**：现场网络或算力不足。
- **应对**：优先本地推理方案，演示前在目标设备上压测通过。
